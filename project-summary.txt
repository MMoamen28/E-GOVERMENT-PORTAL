# E-Government Portal: ID Renewal Module - Project Summary

## What The Project Is About
We are building an E-Government Portal that allows Egyptian citizens to renew their national ID online. The system has multiple microservices working together:
- Citizens can submit an ID renewal request with their name and national ID
- The system validates the name using GoRules business rules engine
- If the name is valid, the request is saved to PostgreSQL database
- A workflow is started in Flowable BPMN engine for supervisor review
- Supervisors can approve or reject the request
- Citizens can check their request status at any time

## Tech Stack
- Backend API: NestJS + TypeScript
- Authentication: Keycloak with JWT tokens
- Business Rules: GoRules BRMS
- Workflow Engine: Flowable REST 6.8.0
- Database: PostgreSQL 15 with TypeORM
- Frontend: HTML + CSS + JavaScript
- Container: Docker + Docker Compose

## What We Have Built
### Step 1 — GoRules Name Validation
Created decision table that rejects empty names, names with numbers, names with special characters, names too short or too long. GoRulesService calls the GoRules API and returns ACCEPT or REJECT with a reason.

### Step 2 — Keycloak Authentication
Set up Keycloak realm egov-portal with client id-renewal-api. Three roles: citizen, supervisor, admin. JWT authentication with NestJS Passport strategy. Users: citizen, supervisor, admin (all password: admin).

### Step 3 — NestJS API
- POST /id-renewal — citizen submits renewal request
- GET /id-renewal — admin/supervisor views all requests
- GET /id-renewal/:id — any role views specific request
- PATCH /id-renewal/:id/complete — supervisor/admin approves or rejects
- POST /id-renewal/deploy — admin deploys BPMN process
- Full Swagger UI at http://localhost:3000/api

### Step 4 — Flowable Workflow
BPMN process with start event, supervisor user task, approve/reject gateway, end event. FlowableService handles deploy, start process, get status, complete task. getSupervisorTasks() returns all pending tasks for supervisor.

### Step 5 — Status Tracking
TypeORM entity RenewalRequestEntity saves requests to PostgreSQL. Status updates from PENDING → APPROVED or REJECTED. rejectionReason field stores the reason when rejected.

### Auth API Module
- POST /auth/login — calls Keycloak and returns JWT token
- POST /auth/register — creates new user in Keycloak and assigns citizen role automatically
- GET /auth/me — returns current user info

### Frontend
- Home page with navigation to ID Renewal module
- Registration page that calls /auth/register to create user in Keycloak
- Login page that calls /auth/login and saves token to localStorage
- ID Renewal page with submit form, status checker, requests list for admin/supervisor, approve/reject buttons

## Problems We Faced and Fixed
- GoRules ESLint errors — Fixed by using catch { } instead of catch (error).
- Keycloak 401 unauthorized — Fixed by adding client_secret to token requests.
- Keycloak account not fully set up — Fixed by turning OFF Temporary password and turning ON Email Verified.
- TypeORM ESLint unsafe errors — Fixed by adding ESLint disable comments.
- form-data import error — Fixed by installing form-data package.
- Flowable BPMN file not found — Fixed by using __dirname.
- Docker build failing — Fixed by exporting KeycloakTokenResponse interface.
- Prettier formatting errors — Fixed by running npm run lint --fix.
- "Conflict" Build Error — Fixed by correcting CompleteTaskDto property mismatch.
